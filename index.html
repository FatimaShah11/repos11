What is GitHub?

Set up a GitHub Account:

  1. Go to https://github.com/join.
  2. Type a user name, your email address, and a password.
  3. Choose Sign up for GitHub, and then follow the instructions.

Create a GitHub repository:
  1. On the GitHub home page, do one of the following:
    • In Your repositories, choose New repository.
    • On the navigation bar, choose Create new (+), and then choose New repository.
  2. In the Create a new repository page, do the following:
    • In the Repository name box, enter CodeDeployGitHubDemo.
    • Select Public.
    • Clear the Initialize this repository with a README check box. You will create a README.md file manually in the next step instead.
    • Choose Create repository.

Conflicts/Errors and their Solutions

1. Merge Conflicts


  • If images or files are already copied or exits in directory then this error will most probably be occur. Make sure if you are uploading file or image online to GitHub account/repository then those files are not supposed to be in your windows working directory/repository folder.
  • If this error occurs delete the ambiguous files then run pull command again.

  • The git stash command takes your uncommitted changes (both staged and unstaged), saves them away for later use, and then reverts them from your working copy.This commands helps you to get out of deadlock position when some merging process are occuring.

  • If updates made from remote repositary are not generated then you won't be able to push your data locally.
  • To resolve such an error run pull command to accept updates then commit your file by using respective git commit command.This will make it easier to push your files or updates to the remote repositary.
